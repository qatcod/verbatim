# Code of Conduct

Verbatim is a small open-source project. Contributions, issues, and discussions are welcome from everyone, regardless of background.

## Expected behavior

- Be respectful and constructive in code reviews, issues, and discussions.
- Assume good faith. Most disagreements come from incomplete context, not bad intent.
- Critique the code, not the person.
- When you disagree, propose an alternative.

## Unacceptable behavior

- Personal attacks, insults, or threats directed at contributors or maintainers.
- Sustained disruption of discussions, issues, or pull requests.
- Public or private harassment of any kind.

## Enforcement

If you encounter behavior that violates this code — in an issue, a pull request, or anywhere else associated with the project — open a confidential channel by emailing the maintainer (see `SECURITY.md` for the address) with details.

Maintainers will investigate, decide on a response, and communicate it. Responses can include: a private conversation, a public clarification, a temporary block, or permanent removal from the project's spaces.

## Scope

This applies to all project spaces: the GitHub repository, issues, pull requests, code reviews, and any associated chat channels. It also applies to contributors representing the project in public.
